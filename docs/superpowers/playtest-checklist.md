# Elemancy 프로토타입 플레이테스트 체크리스트

## 핵심 질문: "조합 디펜스가 손이 가고 재밌는가?"

재미가 확인되기 전에는 2단계(Godot 출시 빌드)로 넘어가지 않는다.

## 실행 방법

```bash
node scripts/dev-server.js   # http://localhost:8000
```

조작: 캔버스 사각형 **안쪽**을 클릭하면 벤치 첫 원소가 그 자리에 배치됨. 우측 "조합 가능" 버튼으로 합성. "다음 웨이브 ▶"로 전투 시작. (디버그: 콘솔에서 `window.game` 으로 상태 확인 가능.)

### 루프 검증
- [ ] 랜덤 원소로 무엇을 만들지 고민하는 재미가 있는가
- [ ] 조합 → 타워 강화의 인과가 체감되는가
- [ ] 역할(swarm/tank/fast) 섞인 웨이브에 공격타입 다양화가 강제되는가
- [ ] 순환 경로를 "어디에 타워를 까느냐"(커버리지)가 의미있는 결정인가

### 밸런스 감각 (수치 조정은 src/config.js만 수정)
- [ ] seg1(1~10): tier1~2로 무난히 넘어가는가 (너무 쉽/어렵지 않은가)
- [ ] 보스(10/20/30/40)가 해당 tier 없이는 확실히 버거운가
- [ ] 몬스터 100 캡이 적절한 압박인가 (너무 빨리/느리게 차는가)
- [ ] 골드 수급 대비 업그레이드/가챠/도박 선택이 의미있는가
- [ ] 보스 토큰(1·3·9·27)이 후반 빈칸 메우기에 유효한가

### 관찰 기록 (친구/커뮤니티 반응)

- 가장 지루/답답한 순간:
- 즉시 그만두게 만든 요소:

## 알려진 프로토타입 한계 (1단계 의도된 범위)
- 아트/사운드/세이브/옵션 없음 (도형 플레이스홀더)
- 배치 입력은 "벤치 첫 원소를 클릭 지점에" 놓는 단순 방식 — 원소 선택 UI는 미구현
- 업그레이드/도박/가챠/보스토큰 사용 UI 버튼 미구현 (로직은 game.js에 완성, 콘솔/후속 UI로 노출 예정)
- 패널 레이아웃은 최소 수준 (좁은 창에서 줄바꿈 어색)
